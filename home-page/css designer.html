CSS Designer (2020–2022)

From 2020 to 2022 my focus turned to visual systems, animations and design-to-code fidelity. As responsive design matured, emphasis shifted to reusable design tokens, consistent spacing systems, dark/light themes, and a component-first styling approach. My work aimed to bridge the gap between design intent and implementation while keeping code scalable.

Core Work

  • Creating and maintaining a design token system for colors, spacing and typography to ensure consistency across products.
  • Building accessible, responsive components (buttons, forms, cards) with clear API and shallow coupling to business logic.
  • Implementing smooth micro-interactions and CSS-driven animations that enhance perceived quality without harming performance.
  • Implementing themes (dark/light and high-contrast) and ensuring visual accessibility across contrast ratios and font sizes.

Tools & Techniques

I used modern CSS features: custom properties (CSS variables) for tokens, grid & flexbox for layouts, and container queries where supported. I also worked with preprocessors and utility-based systems where appropriate to speed up development, while keeping an eye on generated CSS size and runtime performance.

Performance & Accessibility

Animations were implemented in ways that avoid layout thrashing (transform & opacity), and I used prefers-reduced-motion to respect user preferences. All components were audited for contrast and keyboard navigation to meet core accessibility standards.

Outcome

The result was a cohesive visual language that allowed product teams to build interfaces faster with fewer regressions. A small style system I helped build reduced CSS duplication and improved the speed of implementing new screens by providing ready-to-use building blocks.
